# Асинхронные процессы в микросервисных системах — демонстрационный стенд

Программная часть курсовой работы **«Проектирование и реализация
асинхронных процессов в микросервисных системах»**.

Стенд **доменно-нейтрален**: сервисы названы по архитектурной роли (gateway, coordinator,
worker), а «бизнес-логика» абстрактна (этап обработки 1 → 2 → 3). Цель — показать
*механизмы асинхронности*, а не конкретное приложение.

Асинхронность демонстрируется на трёх уровнях глубины:

1. **Рантайм** — `asyncio`: цикл событий, корутины, future/task (неблокирующая конкурентность «с нуля»).
2. **Паттерны** — собственный брокер сообщений (очереди, pub/sub, ack, повторы, dead-letter).
3. **Распределённая система** — те же паттерны на реальном **RabbitMQ** (Outbox, Saga, идемпотентность).

Связующий приём — **порты и адаптеры**: бизнес-код работает против интерфейса `MessageBroker`,
а реализация подменяется (самописный брокер ↔ RabbitMQ). Поэтому стенд запускается
**и без Docker**, и с ним.

## Требования

- Python 3.11+
- (опционально) Docker Desktop + Docker Compose — для полного распределённого режима и экспериментов
- (опционально) Java + PlantUML — для рендеринга диаграмм из `diagrams/*.puml`

## Установка

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev]"
```

> Windows: если команда `python` открывает Microsoft Store, используйте лаунчер `py`
> (как выше). После активации виртуального окружения команды `python`/`pip` уже указывают
> на него.

## Режимы запуска

### Режим A — лёгкий, без Docker (in-memory брокер)

Все сервисы работают как задачи одного цикла событий на самописном брокере; снаружи доступен
REST-интерфейс gateway. Подходит для изучения логики и быстрого запуска.

```powershell
python examples/run_local.py
# затем, например:
#   POST http://127.0.0.1:8000/jobs        — асинхронный приём (202 + job_id)
#   GET  http://127.0.0.1:8000/jobs/{id}   — статус из read-модели
#   POST http://127.0.0.1:8000/jobs/sync   — синхронный baseline
```

### Режим B — распределённый, через Docker (RabbitMQ)

Каждый сервис — отдельный контейнер, обмен через RabbitMQ. Используется для экспериментов и
получения метрик; веб-консоль RabbitMQ — http://localhost:15672 (guest/guest).

```powershell
docker compose -f deploy/docker-compose.yml up --build
```

## Структура репозитория

```

```

## Диаграммы

Исходники — в `diagrams/*.puml`. Рендеринг в PNG (Java не нужна, через сервер PlantUML):

```powershell
python tools/render_diagrams.py   # -> diagrams/png/*.png
```

Альтернативы: расширение **PlantUML** для VS Code (Alt+D), онлайн-редактор
<https://www.plantuml.com/plantuml>.

## Пояснительная записка (.docx по ГОСТ)

Текст глав — в `docs/*.md`. Сборка единого документа:

```powershell
python tools/build_docx.py        # -> docs/build/kursovaya.docx
```

Документ оформлен по ГОСТ (Times New Roman 14, интервал 1.5, поля 30/10/20/20 мм, нумерация
страниц, таблицы, вставленные рисунки, приложение с листингами). После открытия в Word
обновите оглавление: выделите его и нажмите **F9**. Титульный лист содержит заглушки `[___]`
для заполнения.
